# Обязанности Atlas Agent + Revisor

**Atlas Agent** — локальный оркестратор + **локальный ИИ** (GPT4All/Ollama, без ключей) или Claude при наличии ключа.  
**Atlas Revisor** (`run_revisor.py`) — проверка таблицы, авто-правки, мониторинг новых статей и PRIDE.  
Данные только с диска: `data/projects.csv`, `tmt-projects/Projects/`. **Google Sheets не используется.**

См. также: [PLATFORM_FUNCTIONS.md](./PLATFORM_FUNCTIONS.md)

---

## 1. Роль агента

**Куратор TMT-протеомного атласа:** следит за таблицей проектов, согласованностью нормализации с литературой, полнотой Result Files и актуальностью каталога относительно PRIDE/CPTAC/PDC.

**Claude** отвечает за интерпретацию, формулировки выводов, приоритизацию задач и ответы на вопросы исследователя.  
**Python-пайплайн** — за детерминированные проверки (счётчики, правила, загрузка CSV, API PRIDE/PubMed).

## 2. Обязанности (что агент **обязан** делать)

### 2.1 Таблица проектов (`data/projects.csv`)

| # | Обязанность | Как выполняется |
|---|-------------|-----------------|
| 1 | Знать схему колонок и связи между ними | Группы: идентификация, образцы, TMT, нормализация, FASTA, Result Files |
| 2 | Находить пустые и противоречивые поля | Правила целостности (TMT без нормализации, без Result File и т.д.) |
| 3 | Не менять таблицу без явного запроса пользователя | Только отчёты и рекомендации; правки — по команде |

### 2.2 Нормализация и квантификация

| # | Обязанность | Как выполняется |
|---|-------------|-----------------|
| 4 | Фиксировать `Normalization Strategy`, `Z-Score`, `Quantification_Format` из таблицы | Парсинг CSV |
| 5 | Сверять с **статьёй** (PMID → Europe PMC) и **PRIDE** (описание PXD) | Автопоиск + Claude: вердикт supported / partial / contradicted / unknown |
| 6 | Объяснять расхождения простым языком | Блок `claude_analysis` в отчёте |
| 7 | Не утверждать «подтверждено», если в Methods нет явной формулировки | Claude следует системному промпту |

### 2.3 Result Files и GitHub

| # | Обязанность | Как выполняется |
|---|-------------|-----------------|
| 8 | Сверять имя файла в таблице с `tmt-projects/Projects/<PXD>/` | `audit_local_files` |
| 9 | Отмечать отсутствие папки, архивы без распаковки, несовпадение имён | Отчёт + рекомендация запустить `rebuild-all.ps1` атласа |
| 10 | Ссылаться на репозитории [TMT](https://github.com/arinaatom-cyber/TMT) и [tmt-projects](https://github.com/arinaatom-cyber/tmt-projects) | Метаданные в config |

### 2.4 Зависимости и «мировая картинка»

| # | Обязанность | Как выполняется |
|---|-------------|-----------------|
| 11 | Строить сводку по типам нормализации, органам, TMT plex | `normalization_landscape`, сводка Claude |
| 12 | Искать **новые** TMT-проекты в PRIDE, которых нет в таблице | PRIDE API (опционально) |
| 13 | Предлагать, что добавить в каталог (PXD, орган, гипотеза релевантности) | Список кандидатов + текст Claude |

### 2.5 Статистическая обработка

| # | Обязанность | Как выполняется |
|---|-------------|-----------------|
| 14 | По дизайну строки предлагать пайплайн: фильтры, limma/Wilcoxon/paired/mixed | `stats_advisor` + уточнение Claude |
| 15 | Учитывать TMT batch, paired, multi-organ, мало N | Правила в промпте |
| 16 | Не подменять биостатистика консультацией «как в статье» без проверки Methods | Оговорка в каждом отчёте |

### 2.6 Отчётность

| # | Обязанность | Как выполняется |
|---|-------------|-----------------|
| 17 | После каждого прогона — `reports/*.json` и `*.md` | `save_report` |
| 18 | Исполнительное резюме на русском (или EN по config) | Claude `executive_summary` |
| 19 | Приоритетный список действий (топ-10) | Claude `action_items` |

## 3. Что агент **не делает** (границы)

- Не отправляет данные в Google Sheets.
- Не скачивает сырые RAW без отдельной команды.
- Не публикует ключи API и не коммитит `.env`.
- Не заменяет полный парсинг матриц (`build-organ-proteome.py` атласа).
- Не ставит медицинских диагнозов — только протеомные метаданные и методы.

## 4. Архитектура (Claude + код)

```
data/projects.csv ──► Python (правила, PRIDE, PubMed)
                           │
                           ▼
                    JSON-сводка для Claude
                           │
                           ▼
              Claude API (platform.claude.com)
                           │
                           ▼
        executive_summary, normalization_review,
        action_items, ответы на ask
                           │
                           ▼
                  reports/atlas_agent_report_*.md
```

**Модель по умолчанию:** `claude-sonnet-4-6` (баланс цена/качество).  
**Ключ:** переменная окружения `ANTHROPIC_API_KEY` ([получить ключ](https://console.anthropic.com/)).

## 5. Команды для пользователя

```powershell
# Полный прогон с ИИ-анализом
python run_agent.py

# Без ИИ (только правила Python)
python run_agent.py --no-ai

# Ревизор платформы
python run_revisor.py audit -v
python run_revisor.py fix --apply
python run_revisor.py scan

# Вопрос по одному проекту
python run_agent.py ask --project PXD012173

# Без PRIDE
python run_agent.py --no-pride-scan
```

---

## 6. Критерии успеха

- Таблица читается целиком без ошибок.
- Для выборки TMT-проектов есть вердикт по нормализации с указанием источника (таблица / PMID / PRIDE).
- В отчёте есть понятный список «что сделать дальше».
- Новые PXD из PRIDE (если включён скан) не дублируют уже известные ID.
